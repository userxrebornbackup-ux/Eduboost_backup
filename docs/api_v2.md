# API V2 Overview

The V2 API is the supported EduBoost runtime.

Operational endpoints:

- `GET /health`
- `GET /ready`
- `GET /api/v2/health/deep`
- `GET /metrics`
- `POST /api/v2/auth/login`
- `POST /api/v2/auth/refresh`
- `GET /api/v2/learners/{learner_id}`
- `GET /api/v2/learners/{learner_id}/mastery`
- `POST /api/v2/lessons/generate`
- `POST /api/v2/study-plans/generate/{learner_id}`
- `GET /api/v2/jobs/{job_id}`
- `GET /api/v2/parents/{guardian_id}/dashboard`
- `GET /api/v2/parents/{guardian_id}/export`
- `DELETE /api/v2/learners/{learner_id}`

The V2 slice includes:

- Redis-backed job polling for long-running AI work
- daily quotas by subscription tier
- semantic lesson caching
- strict LLM schema validation
- append-only audit logging
- POPIA consent gating and erasure flows

## Authentication

All protected endpoints require a JWT bearer token obtained from `/api/v2/auth/login`.

```http
Authorization: Bearer <access_token>
```

Tokens expire after **15 minutes**.  Use `/api/v2/auth/refresh` with a valid
refresh token (7-day lifetime) to obtain a new access token.

---

## Response Envelope

Every V2 endpoint returns the same envelope structure:

```json
{
  "data":  <payload or null>,
  "error": <error object or null>,
  "meta":  {
    "api_version": "v2",
    "request_id": "<uuid>"
  }
}
```

`data` and `error` are mutually exclusive: exactly one is non-null.

### Success

```json
{
  "data": { "id": "lrn_abc", "name": "Naledi Dlamini" },
  "error": null,
  "meta": { "api_version": "v2", "request_id": "a1b2c3" }
}
```

### Paginated success

```json
{
  "data": [ ... ],
  "error": null,
  "meta": {
    "api_version": "v2",
    "request_id": "a1b2c3",
    "pagination": {
      "total": 120,
      "page": 1,
      "page_size": 20,
      "pages": 6
    }
  }
}
```

### Error

```json
{
  "data": null,
  "error": {
    "code": "not_found",
    "message": "Learner lrn_xyz does not exist.",
    "field_errors": null,
    "hint": null
  },
  "meta": { "api_version": "v2", "request_id": "a1b2c3" }
}
```

See `docs/error_contract.md` for the full list of error codes.

## Health & Readiness

```http
GET /health
```
Returns `200 OK` with `{"status": "ok"}` when the process is alive.

```http
GET /ready
```
Returns `200 OK` when all dependencies (DB, Redis) are reachable.
Returns `503 Service Unavailable` when any dependency is down.

---

## Versioning

See `docs/api_versioning_policy.md` for the full policy.  In brief:

- Non-breaking additions (new fields, new endpoints) are released without a
  version change.
- Breaking changes require a new major version prefix (`/api/v3/...`) and a
  deprecation notice period of at least 90 days.

## OpenAPI Schema

The canonical schema is committed at `docs/openapi.json` and regenerated by:

```bash
make openapi
```

Drift from the committed schema causes the `openapi-contract` CI job to fail.
