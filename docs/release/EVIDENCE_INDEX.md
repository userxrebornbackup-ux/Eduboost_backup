# EduBoost V2 Release Evidence Index

This index is the authoritative map from release claims to readable evidence artefacts.

## Evidence rules

- A claim is not complete unless its evidence artefact exists and can be opened.
- Local pytest evidence is repository-local only; it is not a substitute for CI, staging, legal, or security signoff.
- CI URLs, staging logs, screenshots, and human approvals must be added after those external systems run.
- Historical/generated reports belong in `docs/archive/` unless they are part of the current release package.
