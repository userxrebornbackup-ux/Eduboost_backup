Diagnostics — IRT Engine

Implements the IRT 2-Parameter Logistic model for adaptive ability estimation and item selection.

IRT 2-Parameter Logistic (2PL) adaptive diagnostic engine.

Implements Item Response Theory scoring used to estimate each learner’s latent ability (theta, θ) from their responses to adaptive questions. The 2PL model uses per-item discrimination (a) and difficulty (b) parameters calibrated against the South African CAPS curriculum for grades R–7.

Mathematical model:

\[P(X=1 \mid \theta) = \frac{1}{1 + e^{-a(\theta - b)}}\]

where:

  • \(\theta\) — learner ability estimate (logit scale, typically −4 to +4)

  • \(a\) — item discrimination (steepness of the ICC curve)

  • \(b\) — item difficulty (ability level at which P = 0.5)

Example

Run a full diagnostic cascade:

from app.modules.diagnostics.irt_engine import DiagnosticEngine

engine = DiagnosticEngine()
result = engine.run_gap_probe_cascade(
    learner_grade=4, items=item_bank,
    correct_item_ids={"q1", "q3", "q5"},
)
print(f"θ = {result['theta']:.3f} ± {result['standard_error']:.3f}")
app.modules.diagnostics.irt_engine.p_correct(theta, a, b)[source]

Compute the 2PL probability of a correct response.

Implements the Item Characteristic Curve (ICC):

\[P(\theta) = \frac{1}{1 + e^{-a(\theta - b)}}\]

Parameters are hard-clamped to prevent overflow: a ∈ [0.1, 4.0], b ∈ [−4, 4], θ ∈ [−5, 5].

Parameters:

  • theta (float) – Learner ability estimate on the IRT logit scale.

  • a (float) – Item discrimination parameter (typically 0.5–2.5).

  • b (float) – Item difficulty parameter (typically −3 to +3).

Returns:

Probability of a correct response in [0, 1].

Return type:

float

Example

p = p_correct(theta=0.0, a=1.0, b=0.0)
assert abs(p - 0.5) < 1e-9  # theta == b → P = 0.5
app.modules.diagnostics.irt_engine.fisher_information(theta, a, b)[source]

Compute the Fisher information for a given IRT item.

Measures how much statistical information the item provides about ability around the current theta estimate:

\[I(\theta) = a^2 \cdot P(\theta) \cdot [1 - P(\theta)]\]

Items with high discrimination (a) and difficulty close to theta provide the most information.

Parameters:

  • theta (float) – Learner ability estimate.

  • a (float) – Item discrimination parameter.

  • b (float) – Item difficulty parameter.

Returns:

Fisher information scalar value.

Return type:

float

Example

info = fisher_information(theta=0.0, a=1.5, b=0.0)
assert info > 0  # maximum info when theta == b
app.modules.diagnostics.irt_engine.update_theta_mle(theta, responses, max_iter=20)[source]

Estimate theta using Newton-Raphson maximum likelihood estimation.

Maximises the log-likelihood of the observed response pattern given the 2PL model. Convergence is reached when the update step is smaller than 1e-4 or max_iter is exhausted.

Parameters:

  • theta (float) – Initial ability value to seed the optimisation.

  • responses (list[tuple[IRTItem, bool]]) – List of (item, correct) tuples pairing IRTItem instances with correctness flags.

  • max_iter (int) – Maximum Newton-Raphson iterations (default 20).

Returns:

Estimated theta value clamped to [-4, 4].

Return type:

float

Example

theta = update_theta_mle(0.0, [(item, True), (item2, False)])
assert -4.0 <= theta <= 4.0
class app.modules.diagnostics.irt_engine.DiagnosticEngine[source]

Bases: object

Runs the Gap-Probe Cascade using a real IRT item bank.

The engine estimates learner ability via Expected A Posteriori (EAP) integration, identifies knowledge gaps from missed items, selects the next most informative item via Fisher information, and returns grade-equivalent guidance aligned to the CAPS curriculum.

All public methods are synchronous (CPU-bound maths); database I/O is handled by the calling service layer.

Example

engine = DiagnosticEngine()
result = engine.run_gap_probe_cascade(
    learner_grade=3, items=bank,
    correct_item_ids={"q1", "q3"},
)
print(result["theta"], result["ranked_gaps"])

estimate_theta_eap(responses, *, prior_mean=0.0, prior_sd=1.0, theta_min=-4.0, theta_max=4.0, step=0.1)[source]

Estimate theta via Expected A Posteriori (EAP) integration.

Computes the posterior mean of theta by integrating over a discrete grid with a Gaussian prior:

\[\hat{\theta}_{EAP} = \frac{\sum_k \theta_k \cdot L(\theta_k) \cdot \pi(\theta_k)} {\sum_k L(\theta_k) \cdot \pi(\theta_k)}\]
Parameters:

  • responses (list[tuple[IRTItem, bool]]) – List of (item, correct) tuples.

  • prior_mean (float) – Mean of the Gaussian prior for theta.

  • prior_sd (float) – Standard deviation of the theta prior.

  • theta_min (float) – Minimum theta on the integration grid.

  • theta_max (float) – Maximum theta on the integration grid.

  • step (float) – Grid resolution for theta values.

Returns:

(theta_estimate, standard_error).

Return type:

tuple[float, float]

Example

theta, se = engine.estimate_theta_eap(
    [(item1, True), (item2, False)],
    prior_mean=0.0,
)
assert -4.0 <= theta <= 4.0

map_grade_equivalent(theta, learner_grade)[source]

Convert theta into a grade-equivalent recommendation.

Maps the ability estimate to a recommended CAPS grade level (0–7) by applying a shift based on theta magnitude:

  • θ ≤ −1.8 → −2 grades

  • −1.8 < θ ≤ −0.8 → −1 grade

  • −0.8 < θ < 0.8 → same grade

  • 0.8 ≤ θ < 1.8 → +1 grade

  • θ ≥ 1.8 → +2 grades

Parameters:

  • theta (float) – Estimated learner ability.

  • learner_grade (int) – Current learner grade level (0–7).

Returns:

Recommended grade equivalent clamped to 0–7.

Return type:

int

Example

assert engine.map_grade_equivalent(2.0, 3) == 5
assert engine.map_grade_equivalent(-2.0, 3) == 1

Lessons — LLM Gateway

Provider-agnostic gateway for AI lesson generation with semantic caching and automatic Groq → Anthropic fallback.

Provider-agnostic LLM gateway with automatic fallback.

Abstraction layer over Groq (primary) and Anthropic Claude (fallback) for all AI lesson-generation calls in EduBoost. Real learner UUIDs are never passed to external providers — always use pseudonym_id.

All calls are instrumented with Prometheus metrics via app.core.metrics for latency, token usage, and error tracking.

Example

Generate a lesson completion:

from app.modules.lessons.llm_gateway import LLMGateway

gateway = LLMGateway()
response = await gateway.generate(
    prompt="Explain fractions for Grade 4",
    system="You are a CAPS-aligned tutor.",
)
print(response.content[:200])

Consent — POPIA Service

Full lifecycle management of parental consent records in compliance with the Protection of Personal Information Act.

POPIA parental consent lifecycle service.

Enforces active parental consent before learner data or lesson generation can proceed. All operations are audited for South African POPIA compliance using either a provided AuditRepository or the FourthEstateService fallback.

Every consent state change (grant, revoke, renew, erasure) is recorded in the audit trail for compliance and investigation.

Example

Check and enforce consent before accessing learner data:

from app.modules.consent.service import ConsentService

svc = ConsentService(db)

Learners

Learner profile and archetype management module.

Provides the EtherService (Constitutional Pillar 5 — The Ether) for cold-start onboarding and psychological archetype profiling.

The archetype is assigned on the learner’s first session via a five-question micro-diagnostic, eliminating the legacy 8–10 event lag. Archetype labels follow Kabbalistic naming (Keter, Chokmah, Binah, …) and map to concrete LLM prompt-tone modifiers for personalised lesson delivery.
