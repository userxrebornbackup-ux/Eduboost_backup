# Post-Baseline Roadmap Architecture Contract

## Purpose

This contract defines roadmap governance after the production-readiness baseline.

## Required Roadmap Controls

- baseline boundary register
- post-baseline roadmap register
- deferred scope register
- roadmap dependency register
- graduation criteria
- roadmap review cadence
- post-baseline risk register
- owner assignment
- prioritization model
- explicit external/manual dependencies

## Required Horizons

- now
- next
- later
- parked

## Required Categories

- product
- engineering
- security
- privacy
- AI safety
- infrastructure
- data
- operations
- commercial
- research

## Boundary

This contract records repository-side roadmap governance readiness. It does not approve future scope or complete deferred items.
