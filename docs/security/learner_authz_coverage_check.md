# Learner Authorization Coverage Check

## Purpose

`scripts/check_learner_authz_coverage.py` blocks newly introduced learner-scoped
routes that do not contain an authorization or authentication marker.

## Make Target

```bash
make learner-authz-check
```

The check uses the learner authorization matrix generated by:

```bash
make learner-authz-matrix
```

## Allowlisted Non-Object Routes

The learner-object coverage guard intentionally allowlists these routes:

```text
POST /api/v2/auth/dev-session
POST /api/v2/consent-renewal/trigger-renewal-reminders
GET /api/v2/ether/onboarding/questions
GET /api/v2/gamification/leaderboard
```

They are not direct learner-object routes. They remain visible in the matrix and
can be hardened by separate operational-auth slices if needed.

## Verification

```bash
pytest -c pytest.ini tests/unit/test_check_learner_authz_coverage.py -q --no-cov
make learner-authz-check
```
